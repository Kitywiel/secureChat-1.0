📡 How to use this inbox with Discord, GitHub, etc. & Tor setup guide

Using with any service

Give the address above (__INBOX_ADDRESS__) when signing up for a service. When they send a verification email you will see it appear on this page automatically (if the server has SMTP enabled) or you can paste the confirmation link into this page using the HTTP drop form.

Enabling real inbound email (SMTP)

  • You need a public domain with DNS control.
  • Add an MX record: mail.yourdomain.com. MX 10 <server-public-IP>
  • Start the server with: MAIL_DOMAIN=mail.yourdomain.com SMTP_PORT=25 python -m server
  • Port 25 requires root. Run behind a reverse-proxy or use port 2525 with a firewall NAT rule (iptables -t nat -A PREROUTING -p tcp --dport 25 -j REDIRECT --to-port 2525).

Hosting over Tor

  • The secureChat web UI works perfectly over a Tor hidden service (.onion address).
  • ⚠️ SMTP / email cannot travel over Tor — receiving real email from Discord/GitHub requires a clearnet IP with a proper MX record. Tor hidden services do not have MX-resolvable addresses.
  • Recommended setup: host the server on a VPS with a clearnet IP for SMTP, add a Tor hidden service for the web UI (port 5000 → .onion), and use MAIL_DOMAIN=mail.yourdomain.com pointing at the VPS IP.

Privacy

  • Inboxes are held only in memory and auto-destroyed at the TTL you chose.
  • No messages are stored on disk or in the database.
  • The server only logs the token suffix (last 6 chars) — not your full address.
