# secureChat — Environment Variables Reference

All variables are optional. secureChat works out-of-the-box without a `.env`
file.

## Quick start

Generate a `.env` file from the bundled example and edit it:

```
python run.py --example
```

Then open `.env`, uncomment the lines you want to change, and run:

```
python run.py
```

---

## How to set a variable

Pick any one of the three methods below — they are all equivalent:

| Method | When to use |
|---|---|
| `.env` file (all platforms) | Persistent settings stored next to `run.py` |
| `start_server.bat` (Windows) | Edit the `SET` lines; remove `::` to activate |
| Command-line flag | One-off override: `python run.py --port 8080` |

`.env` and `start_server.bat` values are loaded at startup.  
Command-line flags always win over both.

---

## Server

| Variable | Default | Description |
|---|---|---|
| `PORT` | `5000` | TCP port the server listens on. |
| `HOST` | `127.0.0.1` (Tor) / `0.0.0.0` (no Tor) | Bind interface. |
| `DB_PATH` | `securechat.db` next to `run.py` | Path to the SQLite database file. |

---

## Custom / Persistent URL Paths

secureChat auto-generates these secrets on first run and writes them back into
`start_server.bat` (Windows) or `.env` (other platforms) so the same paths are
reused on every restart.

| Variable | Default | Description |
|---|---|---|
| `CLEARNET_PATH` | auto-generated | Clearnet access URL path (share with users). Should be hard to guess. |
| `ADMIN_PATH` | auto-generated (200 chars) | Admin panel URL path. Very hard to guess. |
| `ADMIN_PASSCODE` | auto-generated (100 chars) | Password for the admin panel. |
| `ADMIN_WEBHOOK_TOKEN` | auto-generated | Secret authorising the incoming webhook. |
| `MESH_TOKEN` | auto-generated | Invite secret for the server-to-server mesh network. |

To rotate a secret, delete its line from `.env` (or `start_server.bat`) and
restart.

## Tor

| Variable | Default | Description |
|---|---|---|
| `NO_TOR` | not set | Set to `1` to skip Tor even when it is installed. |
| `TOR_PATH` | auto-detected | Absolute path to the `tor` binary. |
| `ONION_ADDRESS` | auto-generated by Tor | Your known `.onion` hostname. Set this if you already have a hostname from a previous run or a vanity address so invite links return the correct URL. |

---

## Mesh / Federation

| Variable | Default | Description |
|---|---|---|
| `MESH_JOIN` | not set | Full `/mesh/peer/connect` URL of a remote server to auto-join at startup. |
| `MESH_TOKEN` | auto-generated | Invite secret of the remote server (see above). |

---

## Mail / SMTP

| Variable | Default | Description |
|---|---|---|
| `MAIL_DOMAIN` | not set | Your real mail domain (e.g. `yourdomain.com`). Required to accept inbound email on port 25. Without this, secureChat uses the automatic mail.tm disposable address. |
| `SMTP_PORT` | `25` | SMTP listen port. Use `2525` if not running as Administrator/root. |
| `RELAY_SECRET` | auto-generated | Secret shared with your relay service (Mailgun, SendGrid, Cloudflare, …). Printed to the console at startup. |

---

## Auto-Update

| Variable | Default | Description |
|---|---|---|
| `AUTO_UPDATE` | not set (disabled) | Set to `1` to automatically run `git pull --ff-only` before starting. Only works when launched from a git clone. |

---

## Slow Mode

| Variable | Default | Description |
|---|---|---|
| `SLOW_MODE` | `0` (off) | Set to `1` to start the server with slow mode already active. |
| `SLOW_MODE_DELAY` | `2.0` | Delay in seconds applied to each request while slow mode is active. |

---

## DDoS Protection

| Variable | Default | Description |
|---|---|---|
| `DDOS_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in DDoS guard entirely. |
| `DDOS_REQ_LIMIT` | `200` | Max requests per IP inside the sliding window before the IP is banned. |
| `DDOS_WINDOW_SEC` | `10` | Sliding window length in seconds. |
| `DDOS_BAN_SEC` | `300` (5 min) | How long a banned IP stays blocked in seconds. |
| `DDOS_AUTO_LOCKDOWN_THRESHOLD` | `50` | Number of unique IPs banned within one window that triggers automatic lockdown. |

---

## Spam Protection

| Variable | Default | Description |
|---|---|---|
| `SPAM_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in chat/mail spam filter. |
| `SPAM_MSG_LIMIT` | `20` | Max chat messages per sender inside the sliding window. |
| `SPAM_MSG_WINDOW` | `10` | Chat spam sliding window in seconds. |
| `SPAM_MAIL_LIMIT` | `5` | Max inbound emails per sender inside the sliding window. |
| `SPAM_MAIL_WINDOW` | `60` | Mail spam sliding window in seconds. |

---

## Chat History

| Variable | Default | Description |
|---|---|---|
| `HISTORY_LIMIT` | `100` | Max messages stored and replayed per room. Oldest are pruned automatically. |

---

## mail.tm Integration

| Variable | Default | Description |
|---|---|---|
| `MAILTM_ENABLED` | `1` (enabled) | Set to `0` to disable the automatic mail.tm disposable-address integration. Only applies when `MAIL_DOMAIN` and `RELAY_SECRET` are both unset. |

## Local Mesh (multi-instance on the same machine)

Run multiple secureChat instances on one machine and keep them in sync via the
loopback interface.  Start the hub first, then start each server with both
variables set.

```
python local_mesh.py          # start the hub (port 9000 by default)
LOCAL_MESH_PORT=9000 FILE_STORAGE=storage PORT=5000 python run.py
LOCAL_MESH_PORT=9000 FILE_STORAGE=storage PORT=5001 python run.py
```

| Variable | Default | Description |
|---|---|---|
| `LOCAL_MESH_PORT` | not set (disabled) | Port of the `local_mesh.py` hub on `127.0.0.1`. When set, this instance registers with the hub and chat messages are fanned out to all other registered instances in real time over loopback. |
| `FILE_STORAGE` | not set (temp dirs) | Directory for shared file storage. When set, uploaded files are written here instead of a per-process temp directory, so any instance can serve the download regardless of which URL the uploader used. Set the same path on every instance. |
