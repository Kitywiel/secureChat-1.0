# secureChat â€” start_server.bat Configuration Reference

`start_server.bat` is the Windows one-click launcher for secureChat.
Double-click it to start the server â€” no manual configuration is needed.

All options are **optional**.  To activate a setting, add the flag directly
to the `python run.py` line at the bottom of the file, or set the
corresponding environment variable in a `.env` file placed next to `run.py`.

Example â€” run on port 8080 without Tor:
```
python run.py --port 8080 --no-tor
```

Requirements:
- Python 3.9 or newer (<https://www.python.org/downloads/>)
  Tick "Add python.exe to PATH" during installation.
- Internet connection on first run only (downloads Tor + Python packages;
  subsequent runs work fully offline).

---

## Server

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--port N` | `PORT` | `5000` | TCP port the server listens on. |
| `--host ADDR` | `HOST` | `127.0.0.1` (Tor) / `0.0.0.0` (no Tor) | Bind interface. |
| `--db-path PATH` | `DB_PATH` | `securechat.db` next to `run.py` | Path to the SQLite database file. |

## Custom / Persistent URL Paths

By default secureChat auto-generates these secrets on the first run and
writes the values back into `start_server.bat` as active `SET` lines so
the same paths are reused on every subsequent restart.

You can set a custom value before the first run â€” for example a memorable
name you want to share with specific people:

```
python run.py --clearnet-path MySecureChatServiceName --admin-path MyPrivateAdminArea
```

Rules:
- `--clearnet-path` should be hard to guess (at least 20 random-looking chars).
- `--admin-path` should be very hard to guess (at least 50 random-looking chars).
- `--admin-passcode` is the password that protects the admin panel.
- `--admin-webhook-token` is the secret that authorises the incoming webhook.
- `--mesh-token` is the invite secret for the server-to-server mesh network.

To rotate a secret, delete its `SET` line from `start_server.bat` and restart.

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--clearnet-path PATH` | `CLEARNET_PATH` | auto-generated | Clearnet access URL path. |
| `--admin-path PATH` | `ADMIN_PATH` | auto-generated (200 chars) | Admin panel URL path. |
| `--admin-passcode SECRET` | `ADMIN_PASSCODE` | auto-generated (100 chars) | Admin panel passcode. |
| `--admin-webhook-token TOKEN` | `ADMIN_WEBHOOK_TOKEN` | auto-generated | Admin incoming webhook token. |
| `--mesh-token TOKEN` | `MESH_TOKEN` | auto-generated | Mesh network invite secret. |

## Tor

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--no-tor` | `NO_TOR=1` | not set (Tor enabled) | Skip Tor even if it is installed. |
| `--tor-path PATH` | `TOR_PATH` | auto-detected | Absolute path to the `tor` binary. |
| `--onion-address HOSTNAME` | `ONION_ADDRESS` | auto-generated by Tor | Your known `.onion` hostname. Set this if you already have a hostname (e.g. from a previous run or a custom vanity address) so that invite links and `/api/server-info` return the correct onion URL. Leave unset and Tor will auto-generate one on first run. Example: `--onion-address securechatfbdshafdsyfsdgjafkjadsgjfkadg.onion` |

## Mesh / Federation

Automatically join a remote peer at startup.  Set `--mesh-join` and
`--mesh-token` together.

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--mesh-join URL` | `MESH_JOIN` | not set | Full `/mesh/peer/connect` URL of the remote server. |
| `--mesh-token TOKEN` | `MESH_TOKEN` | auto-generated | Invite secret (see *Custom URL paths* above). |

## Mail / SMTP

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--mail-domain DOMAIN` | `MAIL_DOMAIN` | not set | Your real mail domain (e.g. `yourdomain.com`). Required to accept inbound email directly on port 25. Without this, secureChat uses the automatic mail.tm disposable address. |
| `--smtp-port N` | `SMTP_PORT` | `25` | SMTP listen port. Use `2525` if not running as Administrator. |
| `--relay-secret SECRET` | `RELAY_SECRET` | auto-generated | Secret shared with your relay service (Mailgun, SendGrid, Cloudflare, â€¦). Printed to the console at startup. |

| Flag | Env var | Default | Description |

## Slow Mode

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--slow-mode` | `SLOW_MODE=1` | off | Start the server with slow mode already active. |
| `--slow-mode-delay SECONDS` | `SLOW_MODE_DELAY` | `2.0` | Delay in seconds applied to each request while slow mode is active. |

## DDoS Protection

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--ddos-enabled 0\|1` | `DDOS_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in DDoS guard entirely. |
| `--ddos-req-limit N` | `DDOS_REQ_LIMIT` | `200` | Max requests per IP inside the sliding window before the IP is banned. |
| `--ddos-window-sec N` | `DDOS_WINDOW_SEC` | `10` | Sliding window length in seconds. |
| `--ddos-ban-sec N` | `DDOS_BAN_SEC` | `300` (5 min) | How long a banned IP stays blocked in seconds. |
| `--ddos-auto-lockdown-threshold N` | `DDOS_AUTO_LOCKDOWN_THRESHOLD` | `50` | Number of unique IPs banned within one window that triggers automatic lockdown. |

---

## Spam Protection

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--spam-enabled 0\|1` | `SPAM_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in chat/mail spam filter. |
| `--spam-msg-limit N` | `SPAM_MSG_LIMIT` | `20` | Max chat messages per sender inside the sliding window. |
| `--spam-msg-window N` | `SPAM_MSG_WINDOW` | `10` | Chat spam sliding window in seconds. |
| `--spam-mail-limit N` | `SPAM_MAIL_LIMIT` | `5` | Max inbound emails per sender inside the sliding window. |
| `--spam-mail-window N` | `SPAM_MAIL_WINDOW` | `60` | Mail spam sliding window in seconds. |

| Flag | Env var | Default | Description |

## Mail / mail.tm

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--mailtm-enabled 0\|1` | `MAILTM_ENABLED` | `1` (enabled) | Set to `0` to disable the automatic mail.tm disposable-address integration. Only applies when `MAIL_DOMAIN` and `RELAY_SECRET` are both unset. |
