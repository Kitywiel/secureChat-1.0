# secureChat — start_server.bat Configuration Reference

`start_server.bat` is the Windows one-click launcher for secureChat.
Double-click it to start the server — no manual configuration is needed.

All options are **optional**.  To activate a setting, remove the leading
`:: ` from the `SET` line, set your desired value, save the file, and
double-click to start.

Requirements:
- Python 3.9 or newer (<https://www.python.org/downloads/>)
  Tick "Add python.exe to PATH" during installation.
- Internet connection on first run only (downloads Tor + Python packages;
  subsequent runs work fully offline).

---

## Server

| Variable | Default | Description |
|---|---|---|
| `PORT` | `5000` | TCP port the server listens on. |
| `HOST` | `127.0.0.1` (Tor) / `0.0.0.0` (no Tor) | Bind interface. |
| `DB_PATH` | `securechat.db` next to `run.py` | Path to the SQLite database file. |

---

## Custom / Persistent URL Paths

By default secureChat auto-generates these secrets on the first run and
writes the values back into `start_server.bat` as active `SET` lines so
the same paths are reused on every subsequent restart.

You can set a custom value before the first run — for example a memorable
name you want to share with specific people:

```
SET CLEARNET_PATH=MySecureChatServiceName
SET ADMIN_PATH=MyPrivateAdminArea
```

Rules:
- `CLEARNET_PATH` should be hard to guess (at least 20 random-looking chars).
- `ADMIN_PATH` should be very hard to guess (at least 50 random-looking chars).
- `ADMIN_PASSCODE` is the password that protects the admin panel.
- `ADMIN_WEBHOOK_TOKEN` is the secret that authorises the incoming webhook.
- `MESH_TOKEN` is the invite secret for the server-to-server mesh network.

To rotate a secret, delete its `SET` line and restart.

| Variable | Default | Description |
|---|---|---|
| `CLEARNET_PATH` | auto-generated | Clearnet access URL path. |
| `ADMIN_PATH` | auto-generated (200 chars) | Admin panel URL path. |
| `ADMIN_PASSCODE` | auto-generated (100 chars) | Admin panel passcode. |
| `ADMIN_WEBHOOK_TOKEN` | auto-generated | Admin incoming webhook token. |
| `MESH_TOKEN` | auto-generated | Mesh network invite secret. |

## Tor

| Variable | Default | Description |
|---|---|---|
| `NO_TOR` | not set (Tor enabled) | Set to `1` to skip Tor even if it is installed. |
| `TOR_PATH` | auto-detected | Absolute path to the `tor` binary. |
| `ONION_ADDRESS` | auto-generated by Tor | Your known `.onion` hostname. Set this if you already have a hostname (e.g. from a previous run or a custom vanity address) so that invite links and `/api/server-info` return the correct onion URL. Leave unset and Tor will auto-generate one on first run. Example: `SET ONION_ADDRESS=securechatfbdshafdsyfsdgjafkjadsgjfkadg.onion` |

## Mesh / Federation

Automatically join a remote peer at startup.  Set `MESH_JOIN` and
`MESH_TOKEN` together.

| Variable | Default | Description |
|---|---|---|
| `MESH_JOIN` | not set | Full `/mesh/peer/connect` URL of the remote server. |
| `MESH_TOKEN` | auto-generated | Invite secret (see *Custom URL paths* above). |

---

## Mail / SMTP

| Variable | Default | Description |
|---|---|---|
| `MAIL_DOMAIN` | not set | Your real mail domain (e.g. `yourdomain.com`). Required to accept inbound email directly on port 25. Without this, secureChat uses the automatic mail.tm disposable address. |
| `SMTP_PORT` | `25` | SMTP listen port. Use `2525` if not running as Administrator. |
| `RELAY_SECRET` | auto-generated | Secret shared with your relay service (Mailgun, SendGrid, Cloudflare, …). Printed to the console at startup. |

---

## Auto-Update

| Variable | Default | Description |
|---|---|---|
| `AUTO_UPDATE` | not set (disabled) | Set to `1` to automatically run `git pull --ff-only` before starting. Only works when launched from a git clone. |

---

## Slow Mode

| Variable | Default | Description |
|---|---|---|
| `SLOW_MODE` | `0` (off) | Set to `1` to start the server with slow mode already active. |
| `SLOW_MODE_DELAY` | `2.0` | Delay in seconds applied to each request while slow mode is active. |

---

## DDoS Protection

| Variable | Default | Description |
|---|---|---|
| `DDOS_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in DDoS guard entirely. |
| `DDOS_REQ_LIMIT` | `200` | Max requests per IP inside the sliding window before the IP is banned. |
| `DDOS_WINDOW_SEC` | `10` | Sliding window length in seconds. |
| `DDOS_BAN_SEC` | `300` (5 min) | How long a banned IP stays blocked in seconds. |
| `DDOS_AUTO_LOCKDOWN_THRESHOLD` | `50` | Number of unique IPs banned within one window that triggers automatic lockdown. |

---

## Spam Protection

| Variable | Default | Description |
|---|---|---|
| `SPAM_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in chat/mail spam filter. |
| `SPAM_MSG_LIMIT` | `20` | Max chat messages per sender inside the sliding window. |
| `SPAM_MSG_WINDOW` | `10` | Chat spam sliding window in seconds. |
| `SPAM_MAIL_LIMIT` | `5` | Max inbound emails per sender inside the sliding window. |
| `SPAM_MAIL_WINDOW` | `60` | Mail spam sliding window in seconds. |

---

## Chat History

| Variable | Default | Description |
|---|---|---|
| `HISTORY_LIMIT` | `100` | Max messages stored and replayed per room. Oldest are pruned automatically. |

---

## Mail / mail.tm

| Variable | Default | Description |
|---|---|---|
| `MAILTM_ENABLED` | `1` (enabled) | Set to `0` to disable the automatic mail.tm disposable-address integration. Only applies when `MAIL_DOMAIN` and `RELAY_SECRET` are both unset. |
