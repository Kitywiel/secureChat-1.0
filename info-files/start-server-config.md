# secureChat — Configuration Reference

This file documents every configuration option for secureChat.

---

## How to configure

There are three equivalent ways to set an option — pick whichever suits you:

### 1. `start_server.bat` (Windows, recommended)

Open `start_server.bat` in a text editor.  Each option is listed as a
commented-out `SET` line.  Remove the leading `::` to activate it:

```bat
:: before (inactive)
:: SET PORT=5000

:: after (active)
SET PORT=5000
```

### 2. `.env` file (all platforms)

Copy `.env.example` to `.env`, then uncomment the lines you want:

```
# before (inactive)
# PORT=5000

# after (active)
PORT=5000
```

Existing shell environment variables are **never** overwritten by `.env`.

### 3. Command-line flags (all platforms)

Pass flags directly to `python run.py`:

```
python run.py --port 8080 --no-tor
```

Command-line flags take precedence over `.env` and `SET` values.

---

## Server

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--port N` | `PORT` | `5000` | TCP port the server listens on. |
| `--host ADDR` | `HOST` | `127.0.0.1` (Tor) / `0.0.0.0` (no Tor) | Bind interface. |
| `--db-path PATH` | `DB_PATH` | `securechat.db` next to `run.py` | Path to the SQLite database file. |

## Custom / Persistent URL Paths

By default secureChat auto-generates these secrets on the first run and
writes the values back into `start_server.bat` as active `SET` lines so
the same paths are reused on every subsequent restart.

You can set a custom value before the first run — for example a memorable
name you want to share with specific people:

```
python run.py --clearnet-path MySecureChatServiceName --admin-path MyPrivateAdminArea
```

Rules:
- `--clearnet-path` should be hard to guess (at least 20 random-looking chars).
- `--admin-path` should be very hard to guess (at least 50 random-looking chars).
- `--admin-passcode` is the password that protects the admin panel.
- `--admin-webhook-token` is the secret that authorises the incoming webhook.
- `--mesh-token` is the invite secret for the server-to-server mesh network.

To rotate a secret, delete its `SET` line from `start_server.bat` (or its
line from `.env`) and restart.

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--clearnet-path PATH` | `CLEARNET_PATH` | auto-generated | Clearnet access URL path. |
| `--admin-path PATH` | `ADMIN_PATH` | auto-generated (200 chars) | Admin panel URL path. |
| `--admin-passcode SECRET` | `ADMIN_PASSCODE` | auto-generated (100 chars) | Admin panel passcode. |
| `--admin-webhook-token TOKEN` | `ADMIN_WEBHOOK_TOKEN` | auto-generated | Admin incoming webhook token. |
| `--mesh-token TOKEN` | `MESH_TOKEN` | auto-generated | Mesh network invite secret. |

## Tor

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--no-tor` | `NO_TOR=1` | not set (Tor enabled) | Skip Tor even if it is installed. |
| `--tor-path PATH` | `TOR_PATH` | auto-detected | Absolute path to the `tor` binary. |
| `--onion-address HOSTNAME` | `ONION_ADDRESS` | auto-generated by Tor | Your known `.onion` hostname. Set this if you already have a hostname (e.g. from a previous run or a custom vanity address) so that invite links and `/api/server-info` return the correct onion URL. Leave unset and Tor will auto-generate one on first run. Example: `--onion-address yourhostname.onion` |

## Mesh / Federation

Automatically join a remote peer at startup.  Set `--mesh-join` and
`--mesh-token` together.

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--mesh-join URL` | `MESH_JOIN` | not set | Full `/mesh/peer/connect` URL of the remote server. |
| `--mesh-token TOKEN` | `MESH_TOKEN` | auto-generated | Invite secret (see *Custom URL paths* above). |

## Mail / SMTP

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--mail-domain DOMAIN` | `MAIL_DOMAIN` | not set | Your real mail domain (e.g. `yourdomain.com`). Required to accept inbound email directly on port 25. Without this, secureChat uses the automatic mail.tm disposable address. |
| `--smtp-port N` | `SMTP_PORT` | `25` | SMTP listen port. Use `2525` if not running as Administrator. |
| `--relay-secret SECRET` | `RELAY_SECRET` | auto-generated | Secret shared with your relay service (Mailgun, SendGrid, Cloudflare, …). Printed to the console at startup. |

## Auto-Update

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--auto-update` | `AUTO_UPDATE=1` | not set (disabled) | Automatically run `git pull --ff-only` before starting. Only works when launched from a git clone. |

## Slow Mode

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--slow-mode` | `SLOW_MODE=1` | off | Start the server with slow mode already active. |
| `--slow-mode-delay SECONDS` | `SLOW_MODE_DELAY` | `2.0` | Delay in seconds applied to each request while slow mode is active. |

## DDoS Protection

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--ddos-enabled 0\|1` | `DDOS_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in DDoS guard entirely. |
| `--ddos-req-limit N` | `DDOS_REQ_LIMIT` | `200` | Max requests per IP inside the sliding window before the IP is banned. |
| `--ddos-window-sec N` | `DDOS_WINDOW_SEC` | `10` | Sliding window length in seconds. |
| `--ddos-ban-sec N` | `DDOS_BAN_SEC` | `300` (5 min) | How long a banned IP stays blocked in seconds. |
| `--ddos-auto-lockdown-threshold N` | `DDOS_AUTO_LOCKDOWN_THRESHOLD` | `50` | Number of unique IPs banned within one window that triggers automatic lockdown. |

---

## Spam Protection

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--spam-enabled 0\|1` | `SPAM_ENABLED` | `1` (enabled) | Set to `0` to disable the built-in chat/mail spam filter. |
| `--spam-msg-limit N` | `SPAM_MSG_LIMIT` | `20` | Max chat messages per sender inside the sliding window. |
| `--spam-msg-window N` | `SPAM_MSG_WINDOW` | `10` | Chat spam sliding window in seconds. |
| `--spam-mail-limit N` | `SPAM_MAIL_LIMIT` | `5` | Max inbound emails per sender inside the sliding window. |
| `--spam-mail-window N` | `SPAM_MAIL_WINDOW` | `60` | Mail spam sliding window in seconds. |

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--history-limit N` | `HISTORY_LIMIT` | `100` | Max messages stored and replayed per room. Oldest are pruned automatically. |

## Local Mesh (multi-instance on the same machine)

Run multiple secureChat instances on the same machine and keep them in sync
over the loopback interface.  Start `local_mesh.py` once (in its own window or
as a background process), then start each server instance with both options set
to the same values.

```
python local_mesh.py                                        # start the hub
python run.py --local-mesh-port 9000 --file-storage storage --port 5000
python run.py --local-mesh-port 9000 --file-storage storage --port 5001
```

| Flag | Env var | Default | Description |
|---|---|---|---|
| `--local-mesh-port PORT` | `LOCAL_MESH_PORT` | not set (disabled) | Port of the `local_mesh.py` hub on `127.0.0.1`. When set, chat messages are synced to all other registered instances in real time over loopback. |
| `--file-storage PATH` | `FILE_STORAGE` | not set (temp dirs) | Shared directory for uploaded files. When set, any instance can serve any upload — downloads work regardless of which URL the client used. Set the same path on every instance. |
| `--server-name NAME` | `SERVER_NAME` | not set | Human-readable display name for this instance shown in the admin panel cluster table. |
| `--main-server` | `MAIN_SERVER=1` | not set | Mark this instance as the primary/head node. Non-main cluster members will redirect admin-panel visitors here. |
| `--mesh-evict-sec N` | `MESH_EVICT_SEC` | `120` | Seconds without a successful stats poll before the local mesh hub evicts the instance as stale. |

## Requirements

- Python 3.9 or newer — <https://www.python.org/downloads/>
  (Tick "Add python.exe to PATH" during installation on Windows.)
- Internet connection on **first run only** — downloads Tor and Python
  packages automatically; subsequent runs work fully offline.
